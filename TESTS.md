# Описание автоматических тестов (Light Way)

Этот файл коротко объясняет, **что именно проверяют** unit-тесты в проекте `LightWay.Tests` и **чего тесты не касаются**. Так проще защищать работу: видно охват логики без чтения кода тестов.

## Как запустить

Из папки решения (где лежит `LightWay.sln`):

```text
dotnet test
```

## Что проверяется тестами

### Очки (`GameScoreTests`)

- При одинаковом лимите ходов **меньше сделанных поворотов** даёт **больше очков**, чем много поворотов.
- Если на уровне включён **таймер**, то **оставшееся время** добавляет бонус к очкам (по сравнению с уровнем без таймера).
- Очки **не падают ниже минимального порога** (даже при «перекрученных» входных данных).

### Уровни (`LevelTests`)

- В игре ровно **5** встроенных уровней (`TotalLevels`).
- Уровни **1–5** успешно загружаются, у каждого есть лимит поворотов (больше нуля) и неотрицательный лимит времени.
- Фабрика `FromDigitMap` **выбрасывает исключение**, если карта не размером 10×10 (защита от ошибок).

### Клетка и зеркало (`CellTests`)

- Поворот зеркала **меняет «\» на «/»** и обратно.
- Флаг «это зеркало» **истинен только** для двух типов зеркал.

### Сброс очков при проигрыше (`GameEngineScoreResetTests`)

- Если закончились **ходы** или **время**, суммарные очки **`TotalScore` сбрасываются в 0** (даже если до этого в тесте искусственно выставили большое значение).

### Проходимость встроенных уровней (`LevelSolvabilityProbe`)

- Для уровней **1–5** перебираются все комбинации ориентаций зеркал; проверяется, что **хотя бы одна** комбинация приводит луч в приёмник (уровень не «сломан»).

### Игровой движок (`GameEngineTests`, `GameEngineInteractionTests`)

- **Прямой путь** без зеркал: луч сразу доходит до приёмника, путь содержит клетку приёмника.
- **Лимит ходов**: если за отведённое число поворотов цель не достигнута — фиксируется **проигрыш**.
- **Лимит времени**: по истечении секунд — **проигрыш**, даже если ходы ещё есть.
- **Победа после поворота**: на маленькой карте одним кликом зеркало переводится в нужное положение, начисляются **очки**.
- **Сброс уровня** снимает проигрыш и обнуляет счётчик поворотов.
- После проигрыша **лишние клики не тратят** ходы.
- Переход на следующий уровень **увеличивает номер**, пока уровни не закончились.
- Клик по **пустой клетке** не увеличивает число поворотов.
- На **последнем** уровне кнопка «логического» перехода дальше не существует: `NextLevel` возвращает `false`.

## Что тестами не покрыто (осознанно)

- **Отрисовка Windows Forms** (цвета, шрифты, отсутствие мерцания, двойная буферизация) — это проверяется глазами при запуске.
- **Точное поведение таймера WinForms** (интервал 1000 мс, старт/стоп вместе с окном) — в тестах вызывается только метод `TickTimer`, который крутит логику на **одну секунду**.
- **Сохранение прогресса в файл**, **звук**, **цветные фильтры** из перспективного списка README — в текущей версии игры **не реализованы**, поэтому отдельных тестов нет.
- **Полный проход всех пяти встроенных уровней руками** (каждый решается за разумное число ходов) — это геймдизайн; автоматический «бот» для прохождения не писался, чтобы не усложнять учебный проект.

## Краткий вывод

Тесты закрывают **игровую логику**: трассировка луча, лимиты ходов и времени, начисление очков, сброс, переход уровней и базовые операции с зеркалом. Визуальная часть и функции «на будущее» из README проверяются описанием выше, а не автотестами.
